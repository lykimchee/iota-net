For more details on our product, please watch our video here.

1. Hardware

The main central routing device, the IoTa Net Hub, can be constructed by either an Arduino or a Raspberry Pi. The IoTa Net Hub connects to the user's WiFi network, searches for available IoT devices, then creates a Personal Area Network for the user. To communicate with our software application, all devices connected to the IoTa Net Hub will send and receive data through the MQTT (Message Queuing Telemetry Transport) Protocol.

2. Software

The supplementary mobile application will allow users to manually search for and connect their IoT devices. It will also collect a variety of different data statistics, including but not limited to device status, power usage, network traffic, and temperature. These statistics will be displayed in a graph format, color-coded to indicate the severity level of a potential security threat. Underneath the graphs are security analytics for each respective statistic. The potential threats and color shades will be determined from a machine learning algorithm that acts as an anomaly-based intrusion detection system. For instance, if the network traffic is colored dark red, it would indicate the current network traffic greatly differs from the measured baseline. The user always has the option to review and manually override the threat if deemed a false positive as well. Lastly, under each statistic will be helpful tips to secure their device, such as recommending a reconfiguration of WiFi settings when the machine learning algorithm suspects an intrusion.

3. User Manuals

The application will also consist of a search page, developed with web scraping algorithms. Here, the user will be able to find a trustworthy manual for their desired IoT device.
